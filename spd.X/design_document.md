# プロジェクト設計書: spdm (PIC Speedometer System)

## 1. 概要
本プロジェクトは、PIC16F1827 マイクロコントローラを使用したデジタル・アナログ併用型スピードメーターシステムです。車輪速センサからのパルス信号を解析して時速を算出し、ステッピングモータによる指針表示と、I2C接続のLCDによる走行距離（ODO）および数値速度表示を行います。

## 2. ハードウェア構成
- **MCU**: PIC16F1827 (内部クロック 32MHz)
- **ステッピングモータ**: X27.168 (3ステップ/度、最大駆動角315度)
- **表示器**: ST7032i 搭載 I2C LCD (8文字x2行)
- **センサ入力**: コンパレータ1 (RA3/DAC入力比較) によるパルス検出
- **電源制御**: 自己保持回路 (RB3) およびキーON信号監視 (RB0)

## 3. ファイル構成
- `source.c`: メインロジック、計算アルゴリズム、割り込みハンドラ
- `app_config.h`: 速度域ごとの補正係数や表示文字列の設定
- `skI2Clib.c / .h`: I2C 通信（MSSP1）ライブラリ
- `skI2CLCDlib.c / .h`: ST7032i 制御ライブラリ

## 4. source.c の詳細仕様

### 4.1 速度計算ロジック (`lambda_to_step`)
パルス周期（$\lambda$）をステッピングモータのステップ数に変換します。
- **変換式**: `target_step = (stpfunc * LONE / lbd / 100)`
  - `LONE`: 1km/h 時のカウント値定数。
  - `stpfunc`: メーター盤面の非線形性を補正するための係数。
- **非線形補正**: 
  - メーターの目盛りに合わせるため、`lbd`（周期）の値によって速度域を判定。
  - `app_config.h` で定義された `STPFUNC_SPEED_XXX` 係数を切り替えて、物理的な指針位置を微調整します。

### 4.2 特殊表示シーケンス（起動・終了時）
通常走行時以外の表示処理です。メッセージ内容は `app_config.h` で定義されており、コードを修正せずにメッセージの変更が可能です。**LCDのハードウェア仕様（8文字x2行）に基づき、各メッセージ（`LCD_xxx_LINE_x`）は最大8文字の固定長として定義する必要があります。**
- **起動時メッセージ**: `initialize_system` 内で I2C LCD 初期化直後に実行されます。
  - 1行目: `LCD_OPENING_LINE1` を座標(0,0)に表示。
  - 2行目: `LCD_OPENING_LINE2` を座標(0,1)に表示。
- **終了時メッセージ**: `shutdown` 内で EEPROM 書き込みの直前に実行されます。
  - `LCD_Clear` 実行後、`LCD_SHUTDOWN_LINE1` (0,0) と `LCD_SHUTDOWN_LINE2` (0,1) を表示します。これにより、電源が完全に落ちるまでの間、ユーザーに終了処理中であることを通知します。

### 4.3 LCD表示処理 (`LcdUpdate`)
限られた処理能力で効率的に表示を更新します。

- **速度算出**: パルス周期から `時速 = LONE / lbuf` を計算。
- **文字列構築アルゴリズム**:
  1. `odo`（総走行距離）と算出速度を、下位桁から順に `temp % 10` を用いて文字配列に格納。
  2. 速度が 0 や低速時に不要な '0' を表示しないようスペース置換処理を実行。
  3. 格納された配列を反転（Reverse）させ、正しい順序の文字列を作成。
- **表示レイアウト**:
  - 1行目: "ODO  SPD"
  - 2行目: "[走行距離]  [速度]" (例: "12345  40")

### 4.4 割り込み処理 (`isr`)
複数のタイマーと周辺機能を高精度に同期させます。

- **C1IF (コンパレータ割込)**: 
  - パルス入力を検知。入力信号の矩形波が不安定な場合を考慮し、立ち上がりと立ち下がりの時間を平均化（`buf >> 1`）して `g_lambda` を更新します。
  - パルス数をカウントし、定数 `37800`（1km相当のパルス数）に達するごとに `odo` をインクリメントします。
- **TMR1IF (タイムアウト)**:
  - 一定時間パルスが来ない場合、車両停止（0km/h）と判断し、`g_lambda` を `LZERO` に設定。
- **TMR2IF (モータ制御用)**:
  - `rotate()` 関数を呼び出し、物理的な針の移動を行います。
- **TMR4IF (LCDリフレッシュ制御)**:
  - 表示のチラつきを抑えるため、カウンタ `upd` を減算し、一定周期でのみ LCD 更新を許可します。

### 4.5 モータ駆動制御 (`rotate`)
X27 ステッピングモータを滑らかに動かすための「ルックアヘッド加減速」を実装しています。

- **6相励磁制御**: `MTHLLH()` 等のマクロを使用して Aポートを出力。X27 の仕様に基づいた 6段階の相切り替えを行います。
- **ダイナミック・タイマー制御**: 
  - `l_acceleration_mode` に応じて `PR2`（タイマー周期）を切り替えます。
  - モード 0（高速）からモード 12（低速：4000us）まで段階的に速度を変化させます。
- **ルックアヘッド（先読み）バッファ**:
  - `dir_buffer` に将来の移動方向を格納。
  - 目的地の直前で「移動方向が変わる」または「停止する」ことを事前に検知し、オーバーシュートを防ぐために減速（`l_acceleration_mode` の増加）を行います。

### 4.6 システムシーケンス

#### 起動時 (`initialize_motor`)
1. **原点出し (Homing)**: 
   - モータを反時計回りに最大ステップ数分強制的に回し、物理的なストッパーに当てて 0 点を確定させます（脱調を利用）。
2. **オープニングデモ**: 
   - 指針をフルスケールまでスイープさせ、正常動作を確認します。

#### 終了時 (`shutdown`)
1. **指針復帰**: 針を 0km/h の位置へ戻します。
2. **データ永続化 (EEPROM)**:
   - `odo` と `pulse`（km未満のカウント）を EEPROM に書き込みます。
   - **寿命対策**: `EEPROM_READ` を行い、現在の値と異なる場合のみ `EEPROM_WRITE` を実行することで、フラッシュメモリの劣化を抑えます。
3. **自己保持解除**: `bat_sig` (RB3) を LOW にし、電源を完全に遮断します。

## 5. 通信・保存データ仕様

### I2C 通信
- 通信速度: 約 400kHz (32MHz 動作時)
- スレーブアドレス: `0x3E` (LCD)

### EEPROM マップ
| アドレス | 内容 |
| :--- | :--- |
| 0x00 - 0x02 | ODO 走行距離 (24bit相当) |
| 0x03 - 0x04 | 内部パルス累計 (16bit) |
